## 👋 Who Am I?

I'm **Himanshu Kumar** - a **Senior Automation QA Engineer** from Noida, India, actively transitioning into **DevOps Engineering**.

I spent 5+ years making sure software *doesn't break* - writing test frameworks, CI pipelines, and automation suites that run thousands of validations before a single line reaches production. Now I'm channeling that same obsession with reliability into the infrastructure layer: building pipelines, containerizing workloads, provisioning cloud infrastructure, and making deployments something you look forward to, not dread.

> *Most DevOps engineers learn to care about quality. I already do - from the other side of the wall.*

## 💡 My Unique Edge: QA → DevOps

Most DevOps engineers focus on *speed of delivery*. I bring **quality thinking into the pipeline itself**:

| QA Mindset I Bring | How It Helps in DevOps |
|---|---|
| 🔍 Failure-mode thinking | I design pipelines that handle edge cases, not just happy paths |
| 📋 Test coverage awareness | I know what needs validation before code ships to prod |
| 🔄 Regression prevention | I build gates that catch regressions at the pipeline level |
| 🐛 Root cause analysis | Debugging prod issues is my natural habitat |
| 📊 Observability instinct | I've always needed to *know* if something broke — and why |
